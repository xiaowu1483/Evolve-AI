# 运行时目录说明

## 用途
本目录用于存储系统运行时的临时数据、会话状态和沙箱文件。
**注意：本目录下的文件通常不纳入版本控制（除 .gitkeep 外），重启后可能被清理。**

## 目录结构

### 1. active_session/
- **用途**: 存储当前正在进行的对话会话数据。
- **关键文件**:
  - `raw_log.md`: 原始对话日志，包含用户输入和 AI 输出。
  - `context.json`: 当前会话的上下文状态（JSON 格式）。
- **AI 操作规范**:
  - 每次用户输入后，必须追加写入 `raw_log.md`。
  - 会话结束时，由 `SessionMigrator` 自动迁移到 `wiki/06_Raw_Archives/`。
  - 不要手动删除此目录文件。

### 2. sandbox/
- **用途**: 代码沙箱执行区，用于测试 AI 生成的代码。
- **关键目录**:
  - `workspace/`: 临时代码写入区。
  - `test_results.md`: 测试运行结果。
- **安全规范**:
  - 所有在此目录运行的代码必须在隔离环境中执行。
  - 禁止访问 `src/core` 或 `wiki` 目录的写权限。
  - 执行超时限制为 30 秒。

### 3. queue/
- **用途**: 异步任务队列，存储待处理的反思或维护任务。
- **文件格式**: JSON Lines (.jsonl)。
- **处理流程**:
  - `agents/reflection_agent` 将任务写入此处。
  - `main.py` 调度器定期读取并执行。

## 维护
- 定期清理：由 `tools/log_rotator.py` 自动处理。
- 空间限制：总大小不超过 500MB，超出后自动压缩旧文件。
