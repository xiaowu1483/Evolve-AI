# 数据持久化目录说明

## 用途
本目录用于存储系统运行时产生的持久化数据，包括数据库、备份文件、向量索引等。
**重要：本目录下的实际数据文件（.db, .json, .bak）不应提交到 Git 版本控制。**

## 目录结构

### 1. backups/
- **用途**: 存储由 `tools/backup.py` 自动生成的文件备份。
- **内容**: `.bak` 文件，按时间戳命名。
- **维护**: 由 `tools/log_rotator.py` 定期清理旧备份。
- **安全**: 包含敏感代码快照，禁止泄露。

### 2. vector_db/
- **用途**: 存储向量数据库文件 (如 ChromaDB, FAISS)。
- **说明**: 虽然本项目主打 LLM Wiki 模式，但仍保留此目录用于少量嵌入式检索（如快速搜索代码片段）。
- **维护**: 可安全删除重建，数据可从 Wiki 重新索引。

### 3. sqlite.db
- **用途**: 主关系型数据库。
- **内容**: 用户配置、会话元数据、审计日志索引。
- ** schema**: 见 `schema.sql`。

### 4. audit_trail.json
- **用途**: 审计追踪日志。
- **内容**: 记录所有 Wiki 和代码的变更历史。
- **维护**: 由 `tools/audit_trail.py` 写入。

## 安全警告
1. **禁止提交**: 确保 `.gitignore` 配置正确，防止 `.db` 或 `.bak` 文件泄露。
2. **权限控制**: 建议设置目录权限为 `700` (仅所有者可读写)。
3. **加密**: 生产环境中建议对 `sqlite.db` 进行加密 (SQLCipher)。

## 初始化
首次运行时，系统会自动创建 `sqlite.db` 和必要子目录。
如需手动初始化数据库结构，请运行：
