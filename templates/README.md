# templates/README.md

# 模板目录说明

## 用途
本目录存储项目标准模板，供 AI 在创建新文件时复制使用。
**核心原则**：所有新生成的代码和文档必须基于本目录模板，以确保结构一致性。

## 模板列表

### 1. module_template/
- **用途**: 创建新的功能模块 (`src/modules/`)。
- **包含**: `main.py`, `config.yaml`, `README.md`。
- **适用场景**: 需要注册到系统、有独立生命周期的功能。

### 2. skill_template/
- **用途**: 创建轻量级技能脚本。
- **包含**: `skill.py`。
- **适用场景**: 简单工具函数，无需复杂配置。

### 3. wiki_page_template.md
- **用途**: 创建新的 Wiki 文档。
- **适用场景**: 架构文档、协议文档、记忆文档。
- **注意**: 必须包含 Front Matter (元数据头)。

### 4. reflection_log_template.md
- **用途**: 标准化反思日志格式。
- **适用场景**: `agents/reflection_agent` 生成日志时使用。

### 5. config_template.yaml
- **用途**: 创建新的配置文件。
- **适用场景**: `config/` 或模块内部配置。

## AI 操作规范
1. **复制而非重写**: 优先复制模板文件，修改占位符 (如 `${TITLE}`)。
2. **保持结构**: 不要删除模板中的关键注释和结构。
3. **更新索引**: 创建新 Wiki 页面后，必须更新对应目录的 `_Index.md`。
4. **验证配置**: 创建新配置文件后，运行验证工具检查格式。

## 维护
- 当项目架构变更时，需同步更新本目录模板。
- 模板文件本身不纳入版本控制的频繁变更，应保持稳定。
